# 2D Side-Scroller

Реализация части 2D скроллер-игры под Android.

## Сборка и запуск

**Требования:** 
- Android Studio 
- NDK 26.3.11579264
- CMake 3.22.1 или выше
- минимальный SDK 26.

1. Открыть проект в Android Studio
2. Sync Gradle
3. Build & Run на устройстве

Ориентация экрана - ландшафтная (landscape).

## Стек технологий

- **Язык игровой логики:** C++11 (NDK)
- **Графика:** OpenGL ES 2.0
- **Android-часть:** Java (Activity, GLSurfaceView)
- **Сборка:** Gradle + CMake
- **Графические ресурсы:** не используются - все объекты отрисованы примитивами, текст - программным bitmap-шрифтом 8×8

## Геймплей

Игрок управляет самолётом, который находится на 1/3 экрана по горизонтали. Самолёт двигается по вертикали с инерцией, следуя за пальцем на экране. Навстречу из-за правого края вылетают противники. Цель - набрать максимум очков, сбивая врагов.

Столкновение с противником, пулей врага или землёй завершает игру.

## Типы противников

| Тип | Цвет | Поведение |
|---|---|---|
| Птица | Белый | Маленькая, медленная, хаотично меняет высоту. Неуязвима для пуль |
| Бомбардировщик | Синий | Большой, летит прямо. Можно сбить (+50 очков) |
| Истребитель | Красный | Размер как у игрока, стреляет. Можно сбить (+100 очков) |
| Метеорит | Тёмно-красный | Падает сверху, сталкивается с самолётами, но не с птицами |

## Архитектура

Все игровые объекты наследуют от абстрактного класса `GameObject` (позиция, скорость, размер, active-флаг, виртуальные `update(dt)` и `render()`).

Класс `Game` владеет всеми объектами, управляет спавном по таймерам, проверяет столкновения (AABB), считает очки и хранит состояние игры (Playing / GameOver).

`Renderer` инкапсулирует OpenGL: два шейдера (цветные примитивы + текстурный для текста), ортографическая проекция, встроенный bitmap-шрифт.

## Game Loop

Каждый кадр (`onDrawFrame` → `Game::frame()`):

1. Рассчитать deltaTime
2. Обновить таймеры спавна, создать новых врагов
3. Обновить все объекты (`update(dt)`)
4. Проверить столкновения
5. Удалить неактивные объекты
6. Отрисовать всё (`render()`)
7. Отрисовать UI (счёт, Game Over)

## Управление

- **Касание экрана** - самолёт плавно перемещается к высоте пальца
- **Стрельба** - автоматическая, с интервалом

## Нарастание сложности

Количество одновременных противников на экране растёт со временем игры:
- Бомбардировщики: `1 + gameTime * 0.03`
- Истребители: `1 + gameTime * 0.1`
- Птицы: `2 + gameTime * 0.15`
